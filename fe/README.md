#### ✨2. 이슈 목록(메인 화면)

- [x] 목록의 상단에는 열린 이슈, 닫힌 이슈에 따라 탭으로 갯수가 카운팅 되도록 한다. 각 탭을 누르면 열린 이슈의 목록과 닫힌 이슈의 목록이 보여지도록 한다.
- [x] 메인 화면에서는 ‘열린 이슈 목록’이 보여지는 것을 기본으로 한다
- [x] 각 목록에는 아래와 같은 정보들을 표시한다. 해당 값이 없는 경우, 표시하지 않는다.
  - 열린 이슈 아이콘 혹은 닫힌 이슈 아이콘
  - 이슈 제목
  - 레이블
  - 이슈 번호
  - 작성자 및 타임스탬프 정보
  - 마일스톤
- [x] 우상단의 [레이블]버튼, [마일스톤]버튼을 누르면 각각 레이블을 관리하는 화면과 마일스톤을 관리하는 화면으로 이동한다. 이때 페이지 이동시, 화면 새로고침 없이 이동해야 한다.
- [x] [이슈 작성]버튼을 누르면, 새로운 이슈를 생성하는 페이지로 이동한다.

#### 🎨3. 이슈 목록 / 필터

- [x] 필터바는 좌측에 [필터]버튼과 우측에 텍스트폼으로 이루어져 있다.
- [x] [필터]버튼을 누르면 이슈에 대한 검색 조건을 설정하는 팝업이 뜬다.
- [x] 텍스트폼에는 필터 적용 상태가 텍스트 형식으로 표시된다.
- [x] 텍스트 폼의 기본 상태는 is:issue is:open 으로, 오픈된 이슈 목록이 나타난다. 반드시 'is:open…'과 같은 형식으로 표시될 필요는 없으며, 자유롭게 알아볼 수 있도록 표시한다.
- [x] [필터]버튼에서 필터 종류 중 하나를 선택하면 아래와 같은 상태가 되도록 한다.
  - 팝업이 자동으로 닫힘
  - 필터바 텍스트폼에 Typing State/Typed 상태로 내가 선택한 필터의 조건이 입력됨
  - 이슈 목록에는 조건이 적용된 이슈만 보여지도록 함
- [x] 필터 종류를 선택하지 않고 팝업 외 다른 영역을 클릭하면, 필터가 적용되지 않고 팝업이 닫힌다.
- [x] 이슈 목록의 오른쪽 상단에도 [담당자]・[레이블]・[마일스톤]・[작성자]에 대한 필터 조건을 설정할 수 있다.
- [x] 각 팝업에서 원하는 필터를 선택하면 팝업이 닫히면서 해당 값의 조건이 적용된 목록만 필터링 되어 나타난다.
- [x] 아무 값도 선택하지 않고 팝업 외 영역을 클릭하면 필터가 적용되지 않고 팝업이 닫힌다.
- [x] 모든 필터는 함께 조합해서 사용할 수 있다.
      **_단, 담당자의 필터를 적용후 작성자의 필터를 적용하면 담당자의 필터가 적용되는 버그가 발생(이밴트 핸들러 겹침)_**
- [ ] 필터가 적용된 상태일 땐, 다음과 같이 필터바와 이슈 목록 사이에 필터를 초기화 할 수 있는 [현재의 검색 필터 및 정렬 지우기]버튼이 나타난다.
- [ ] 해당 버튼을 누르면 적용된 필터가 초기화 되고, 이슈 목록도 원래대로 오픈된 상태의 모든 이슈가 보이도록 한다.
- [ ] 필터링 후 보여줄 결과 값이 없다면 다음과 같이 표시한다.
- [x] 필터바의 텍스트폼에 있는 텍스트를 모두 지우면 Placeholder에 “Search all issues”가 자동으로 표기되며, 그대로 엔터를 누르면 닫힌 이슈를 포함한 모든 이슈 목록이 나타나도록 한다.

#### 🥑4. 이슈 목록 /다중 선택

- [x] 이슈 목록에는 각 이슈마다 [체크 박스]가 있고, 이를 사용해 이슈를 다중 선택할 수 있다.
- [x] [체크 박스]를 하나라도 클릭하면 이슈 목록의 상단 좌측에는 [카운터]가, 우측에는 [상태 수정]버튼이 보이도록 한다.
- [x] 모든 체크 박스가 해제되면 이슈 목록의 상단이 기본 형태로 돌아가도록 한다.
- [ ] 카운터는 체크 박스에 표시된 이슈의 갯수를 카운팅하며, [카운터의 체크 박스]를 클릭하면 전체 선택 또는 전체 해제로 동작한다.
- [x] [상태 수정]버튼을 누르면 다중 선택한 이슈의 상태를 일괄 수정할 수 있는 팝업이 나타난다.
  - 선택한 이슈 열기
  - 선택한 이슈 닫기
- [x] 팝업에서 수정하려는 상태를 클릭하면 즉시 적용 후 팝업은 닫히고, 변경한 상태가 반영되어 이슈 목록이 업데이트 된다.

#### 5. 새로운 이슈 작성

- [x] 이슈 목록에서 [이슈 작성]버튼을 누르면 새로운 이슈를 작성할 수 있는 페이지로 이동한다.
- [x] 새로운 이슈는 제목만 입력해도 [완료]버튼이 활성화 된다. 이 버튼을 누르면, 작성한 내용을 저장하고 이슈 상세 화면으로 이동한다.
- [x] [완료]버튼 좌측의 [작성 취소]버튼을 누르면 아무런 변화 없이 이슈 목록으로 돌아간다.
- [x] 코멘트 영역에는 내용을 작성하다가 멈추는 경우, 현재 입력된 글자 수가 2초간 나타난 후 사라지는 기능을 구현한다.
- [x] 코멘트 영역 하단의 [파일 첨부하기]버튼을 누르면 이미지를 선택할 수 있는 창이 뜬다.
- [x] 이미지를 선택하면 서버에 업로드 되고, 저장된 이미지 주소가 마크다운 형식으로 코멘트 영역에 추가된다.
- [x] 사이드바에서 해당 이슈의 담당자・레이블・마일스톤을 설정할 수 있다. 각 항목의 [화살표]버튼을 누르면 해당 항목의 선택지를 고를 수 있는 팝업이 뜬다.
- [x] 팝업에서 원하는 선택지를 클릭하면 즉시 사이드바에 표시되도록 한다.

#### 6. 이슈 상세

- [x] 오른쪽 상단에는 이슈의 제목을 수정하는 [제목 편집]버튼과 상태를 변경하는 [이슈 닫기]버튼이 있다.
- [x] [이슈 닫기]버튼을 누르면 해당 이슈의 상태가 보관 처리되고, 해당 버튼은 [다시 열기]버튼으로 변경된다.
- [x] 이때 화면에서 이슈 제목 하단의 상태가 표시되는 레이블만 변경되고, 나머지는 유지한다.
- [x] [제목 편집]버튼을 누르면 제목 부분만 텍스트 인풋 컴포넌트로 변경되고, [제목 편집 → 편집 취소]버튼과 [이슈 닫기 → 편집 완료] 버튼으로 변경되도록 한다.
- [x] 제목 수정 후 [편집 완료]버튼을 누르면 이슈 상세 페이지에 즉시 반영된다.
- [x] 코멘트를 작성하는 부분에는 내용을 입력하지 않으면 [+코멘트 작성]버튼이 활성화 되지 않는다.
      **_내용부분은 살짝 문제있음_**
- [x] 내용 작성 후 [+코멘트 작성]버튼을 누르면 다른 코멘트들과 같은 디자인으로 코멘트가 표시된다.
- [x] 이슈 상세페이지의 코멘트 작성폼은 이슈 생성 페이지의 코멘트 작성폼과 동일하게 기능이 적용되도록 한다(현재 입력된 글자수가 2초 간 나타났다가 사라지는 기능, 파일 첨부 기능)
- [x] 자신이 작성한 코멘트에서 [편집]버튼을 누르면, 코멘트 입력 폼으로 상태가 변경되며 내용을 수정할 수 있게 된다. 내용에 변경 사항이 없으면 [편집 완료]버튼은 활성화 되지 않는다.
- [x] 내용 수정 후 [편집 완료]버튼을 누르면 변경 사항이 반영된 코멘트 디자인으로 돌아가고, [편집 취소]버튼을 누르면 아무 변동 없이 원래 내용으로 돌아간다.
- [x] 이슈 상세페이지의 코멘트 작성폼은 이슈 생성 페이지의 코멘트 작성폼과 동일하게 기능이 적용되도록 한다(현재 입력된 글자수가 2초 간 나타났다가 사라지는 기능, 파일 첨부 기능)
- [x] 작성자가 아니더라도 화면 오른쪽의 담당자・레이블・마일스톤 부분은 이슈 생성 화면과 동일한 동작으로 수정할 수 있다.

#### 7. 레이블 목록

- [x] 좌상단의 [레이블]버튼과 [마일스톤]버튼을 통해 손쉽게 두 페이지를 번갈아 이동할 수 있다.
- [x] 우상단의 [레이블 추가]버튼을 누르면 새로운 레이블을 생성할 수 있다.
- [x] 레이블 목록 상단에 전체 레이블 개수가 표시되며, 각 레이블 목록에는 아래의 내용이 보여진다.
  - 레이블 이름과 색
  - 레이블에 대한 한 줄 설명(없을 경우 표시하지 않는다)
  - [편집]버튼과 [삭제]버튼
- [x] 레이블 목록에서 특정 레이블의 [삭제]버튼을 누르면 “정말 이 레이블을 삭제하시겠습니까?(취소/삭제)”의 알럿창이 뜬다.
- [x] [삭제]버튼을 누르면 알럿창이 닫히며 레이블 목록 화면에도 삭제 내역이 반영된다. [취소]버튼을 누르면 아무런 변동사항 없이 알럿창이 닫힌다.
- [x] 우상단의 [레이블 추가]버튼을 누르면, [레이블 추가]버튼은 비활성화 되고, 새로운 레이블을 생성할 수 있는 영역이 나타난다. 새로운 레이블을 생성하는 영역 내의 [취소]버튼을 누르면 해당 영역이 다시 사라진다.
- [x] 레이블 생성 영역에는 아래의 내용들을 입력할 수 있다.
- [x] 레이블 이름(필수)
  - 레이블에 대한 설명(선택사항)
  - 레이블 배경 색상(필수, 직접 입력하거나, 리프레시 버튼을 눌러 랜덤 컬러를 사용할 수 있음) **_-> 직접입력은 못하게 막아둠_**
  - 레이블의 텍스트 색상(필수, 어두운 색/밝은 색)
- [x] 입력한 내용을 바탕으로 레이블을 미리보기 할 수 있고, 필수 항목이 모두 입력되면 [완료]버튼이 활성화 된다.
- [x] [완료]버튼을 누르면 레이블 생성 영역은 사라지고, 레이블 목록에는 가장 위에 새로운 레이블이 추가된다.
- [x] 레이블 목록에서 특정 레이블의 [편집]버튼을 누르면, 해당 레이블이 편집 영역으로 전환된다.
- [x] 레이블 편집 영역은 레이블 생성 영역과 똑같이 기능하되, [완료 → 편집 완료]버튼으로 보여진다.
- [x] [편집 완료]버튼은 레이블에 변경사항이 있을 경우에만 활성화 되도록 하고, [취소]버튼을 누르면 아무런 변경 없이 기존 레이블 목록으로 전환된다.

#### 8. 마일스톤 목록

- [x] 좌상단의 [레이블]버튼과 [마일스톤]버튼을 통해 손쉽게 두 페이지를 번갈아 이동할 수 있다.
- [x] 우상단의 [마일스톤 추가]버튼을 누르면 새로운 마일스톤을 생성할 수 있다.
- [x] 마일스톤 목록 상단에는 열린 마일스톤과 닫힌 마일스톤의 개수가 카운트되며 각각 버튼으로 동작한다.
- [x] 각 마일스톤 목록에는 아래의 내용이 보여진다.
  - 마일스톤 이름
  - 완료일 일정(없을 경우 표시하지 않는다)
  - 레이블에 대한 설명(없을 경우 표시하지 않는다)
  - 전체 진행률, 해당 마일스톤과 관련된 오픈/클로즈 이슈의 개수 표시
  - [닫기], [편집], [삭제]버튼
- [x] 우상단의 [마일스톤 추가]버튼을 누르면, [마일스톤 추가]버튼은 비활성화 되고, 새로운 마일스톤을 생성할 수 있는 영역이 나타난다. 새로운 마일스톤을 생성하는 영역 내의 [취소]버튼을 누르면 해당 영역이 다시 사라진다.
- [x] 마일스톤 생성 영역에는 아래의 내용들을 입력할 수 있다.
  - 마일스톤 이름(필수)
  - 완료일(선택사항)
  - 설명(선택사항)
- [ ] 완료일은 YYYY. MM. DD 형식으로 입력할 수 있으며, 유효한 형식으로 입력된 값인지 확인한다. 유효하지 않을 경우 Error형식의 디자인으로 표시한다.
      **_->형식외의 예외처리 못함_**
- [x] 완료일과 설명은 선택사항이므로 제목만 입력해도 [완료]버튼이 활성화 된다.[완료]버튼을 누르면 마일스톤 생성 영역은 사라지고, 마일스톤 목록에는 가장 위에 새로운 마일스톤이 추가된다.
- [x] 마일스톤 목록에서 특정 마일스톤의 [편집]버튼을 누르면, 해당 마일스톤이 편집 영역으로 전환된다.
- [x] 마일스톤 편집 영역은 마일스톤 생성 영역과 동일하게 기능하되 [완료]버튼은 [편집 완료]버튼으로 보여진다.
- [x] [편집 완료]버튼은 마일스톤에 변경사항이 있을 경우에만 활성화 되고, [취소]버튼을 누르면 아무런 변경 없이 기존 마일스톤 목록으로 전환된다.
